In this section, two base models for all 9 product will be established. For the continuous data series we are going to build ARIMA or ARIMAX models. For ARIMA models establishment is shown with using the Product ID : 85004 which is face cleanser. More than one year, it has a continuous sales in Trendyol. Therefore, we can use this data set for building ARIMAX model.

# reading data from local
pr7 = read.csv("alldata_item7.csv")
pr7 <- as.data.table(pr7)

# manipulation

pr7 <- pr7[,-c("X","w_day")] #subtract useless columns
pr7 <- mutate(pr7, event_date = ymd(event_date)) # converting event date into datetime object
pr7[, Month:=as.numeric(lubridate::month(event_date,label=F))] #adding month information as a numeric variable 
pr7[, Day:=as.numeric(lubridate::wday(event_date,label=F))] #adding day information as a numeric variable 

sold <- data.table(event_date =pr7$event_date,
                   sold_count = pr7$sold_count) # to built ARIMA models we only need output attribute

First of all for the ARIMA models, we need to decide frequency of the period. We have one year of data for this product. We need to check residuals of the data first.

acf(sold$sold_count, lag.max = 35)

pacf(sold$sold_count, lag.max = 35)

From these plots above, we can see that there is seasonality at lag 14. we are going to decompose data with period 7.

soldts <- ts(rev(pr7$sold_count),  freq = 7, start= c(1,1))
resultweekdec <- decompose(soldts,type= "additive")
plot(resultweekdec$random)
title("Random Term of Weekly  Decomposed Data")

random = resultweekdec$random
trend = resultweekdec$trend
season = resultweekdec$seasonal

Random term looks like zero mean and constant variance despite some outliers. We can detect these outliers with external regressor later. For now, we continue with this decomposed item to build a ARIMA model. We are going to use AIC values of the models to get the best fitted model. Again before starting using auto.arima() function and neighbour search for the parameters. We will check the ACF and PACF plots of random term.

acf(random, na.action = na.pass)

pacf(random, na.action = na.pass)

Here is the fitted and actual values of the ARIMA model:

model <- arima(random, order= c(2,0,4)) # final model with respect to AIC values

modelfit <- random - model$residuals
fitted <- modelfit+trend+season

plot(soldts)
points(fitted, type= "l", col = 2)

Seems like the model fit well. However at peaks and downs there was still mis-fitted points. To improve the model, we can add external regressors.

We are going to look for variables that might be correlated to sold amount and can be used as regressors in the ARIMAX model. To do this, first I am going to use pairplots. ggpairs(pr7, columns = c(4,7,8,10,13 ))
